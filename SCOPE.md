# Scope of Work — Guardrail

Source: **Statement of Work (SOW)** — Viscabulary × Jamaican Kitchen, "Multi-Location Digital
Infrastructure & POS Ecosystem", dated May 15 2026. Fixed price **$3,000 USD**, **45-day** timeline,
**6 locations**, 100% asset ownership transfers to Jamaican Kitchen on final settlement.

## ⛔ THE RULE (read before adding anything)

Before building ANY feature, confirm it maps to a deliverable in **§A / §B / §C** below.

- **In scope** → build it.
- **Not listed / "nice to have" / "while we're here"** → it's **OUT of scope**. Do NOT build it.
  Flag it to the client as a possible **change order** (separate pricing) and wait for a decision.
- **Exclusions (§Out)** are client responsibilities — never absorb them into the build.

Scope creep is the enemy of a fixed-price/fixed-timeline contract. When unsure, treat it as out and ask.

---

## §A — Brand Identity & Visual Guidelines
- [ ] Brand guidelines derived from Jamaican Kitchen's identity (typography, color palette, visual assets, UI/UX standards)
- [ ] Applied consistently across all public web pages AND the internal dashboard

## §B — Custom Website (5-tier architecture, fully responsive)
Goal: drive online orders, catering acquisition, customer engagement. The FIVE portals:
1. [ ] **Home** — landing page: brand identity, core offerings, primary CTAs
2. [ ] **Order Now** — digital ordering interface, routed live to the backend dashboard
3. [ ] **Catering** — intake + informational funnel for corporate/private catering requests
4. [ ] **Menu** — dynamic, visual digital menu, optimized mobile + desktop
5. [ ] **More (Utility Hub)** — secondary nav housing:
   - [ ] Customer Rewards & Loyalty Portal
   - [ ] User Login / Account Management
   - [ ] Mobile App Download Prompts & Routing
   - [ ] FAQ
   - [ ] Contact Us (inquiries + location mapping)
   - [ ] About Us (brand story & heritage)

> Anything not in these 5 portals (e.g. a blog, gift cards, reservations, online payment capture,
> a storefront beyond these) is OUT unless added as a change order.

## §C — Enterprise POS Backend Dashboard (multi-location, all 6 locations)
Operational modules:
- [ ] **Live Order Management** — realtime sync of website orders + **visual/audio alerts** for kitchen staff
- [ ] **Catering Management** — tracking pipeline for advance catering orders
- [ ] **Delivery & Logistics Tracking** — status management for active dispatches
- [ ] **Payment Integration** — overview of cleared transactions + active gateway statuses (overview only; gateway fees are client-paid, see §Out)
- [ ] **Menu Management** — global + per-location control of items, descriptions, pricing
- [ ] **Inventory Management** — realtime stock alerts + ingredient depletion tracking
- [ ] **Analytics & Reporting** — sales, revenue, order-volume reports

Controls:
- [ ] **Multi-location & aggregate** — filter metrics/inventory/orders by one location OR master view of all 6
- [ ] **Role-Based Access Control (RBAC)** — up to **10 user accounts**, three tiers:
  - [ ] **Owner** — full admin + financial oversight, all properties
  - [ ] **Admin** — system operations
  - [ ] **Shop Manager** — restricted to specific assigned location(s)

## §Support — Post-delivery (also contracted)
- [ ] 30-day technical support (bug resolution + stability) after launch
- [ ] Train ONE designated staff member to run the admin panel (no external help center)
- [ ] **AI Troubleshooting Manual** — type an error → get exact resolution steps
- Post-30-day maintenance is a SEPARATE pricing agreement (not free ongoing work)

## §Out — Explicitly OUT of scope (client responsibilities / not included)
These are paid directly by Jamaican Kitchen, NOT absorbed into the agency build:
- Domain registration & renewals
- Cloud hosting / server infrastructure setup
- SSL certificates
- Advanced firewall layers / dedicated security protocols
- Premium third-party software licenses
- Payment **gateway fees**

Also out unless raised as a change order: any feature, page, module, integration, or capability
**not named in §A/§B/§C above**.
